# RacePal — Project Instructions

- Use simple language.
- If in doubt, ask before building (validate the idea first).
- If a task will be very complex and time consuming, say so in advance.
- If you have a suggestion or notice something isn't meeting accessibility requirements, say so.
- Always flag cost implications before building. The goal is to keep development free.
- If a feature can't be done for free, say so directly and offer an alternative with the trade-offs clearly explained.
- Challenge overengineering. This is an MVP — the goal is to check the main concept works, not to build everything perfectly.
- The app must comply with GDPR. Flag any GDPR concerns before building a feature.
- When you identify a new requirement, add it to BACKLOG.md with a size, and tell me about it.
