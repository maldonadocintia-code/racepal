# ⚡ RacePal

A social calendar for UK runners — track races and parkruns, follow friends, and share ⚡ lightning-bolt reviews.

Distributed as a sideloaded APK (no Play Store needed).

---

## Features

- **Google Sign-In** — one-tap auth
- **UK Parkrun directory** — 100+ locations pre-loaded
- **Manual race entry** — add any 5K, 10K, half, marathon, ultra, etc.
- **Going / Attended tracking** — mark your intent or completed runs
- **⚡ Lightning-bolt ratings** — 1–5 bolts instead of stars
- **Written reviews** — with finish time and public/followers-only visibility
- **Follow model** — follow public accounts directly; request to follow private ones
- **Activity feed** — see what people you follow are running and reviewing
- **Profile pages** — public/private toggle, bio, stats

## Setup (one-time)

### 1. Prerequisites

```bash
# Install Flutter SDK (3.19+)
https://docs.flutter.dev/get-started/install/windows

# Install Firebase CLI
npm install -g firebase-tools
firebase login

# Install FlutterFire CLI
dart pub global activate flutterfire_cli
```

### 2. Create Firebase project

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. **Create project** → call it `racepal` (or anything)
3. Enable **Authentication** → Sign-in providers → **Google**
4. Enable **Firestore Database** → Start in production mode
5. Enable **Storage**

### 3. Connect Flutter to Firebase

Inside the `racepal/` folder:

```bash
flutterfire configure
```

- Select your Firebase project
- Tick **Android** only
- This auto-generates `lib/firebase_options.dart` (replaces the stub)
- It also downloads `google-services.json` into `android/app/`

### 4. Deploy Firestore rules and indexes

```bash
firebase deploy --only firestore:rules,firestore:indexes
```

### 5. Install dependencies & run

```bash
flutter pub get
flutter run          # on connected device, or:
flutter build apk --release   # build APK to sideload
```

The APK is output to:
`build/app/outputs/flutter-apk/app-release.apk`

Transfer this to your Android phone and install (you may need to enable *Install unknown apps* in Settings → Security).

## Google Sign-In — Android SHA-1 fingerprint

Firebase requires your app's SHA-1 to allow Google Sign-In on Android.

**Debug key (for development):**
```bash
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
```

Copy the SHA-1 fingerprint and add it in:
Firebase Console → Project settings → Your apps → Android app → Add fingerprint

**Release key:** generate a keystore and repeat the process for the release SHA-1.

## Firestore data model

```
users/{uid}
  displayName, email, photoUrl, bio
  isPublic, followersCount, followingCount, racesCount, createdAt

races/{raceId}
  name, location, type, category (race|parkrun)
  date, website, description, lat, lng
  createdBy, attendeeCount, averageRating, reviewCount

attendances/{userId_raceId}
  raceId, userId, status (going|attended|interested), createdAt

reviews/{reviewId}
  raceId, userId, userName, userPhotoUrl
  rating (1-5), body, finishTime, isPublic, createdAt

follows/{followerUid_followingUid}
  followerUid, followingUid, createdAt

follow_requests/{requesterUid_targetUid}
  requesterUid, targetUid, createdAt

activities/{actId}
  userId, userName, userPhotoUrl
  type (going|attended|review)
  raceId, raceName, rating, reviewBody, createdAt
```
